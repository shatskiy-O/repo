# Архитектура и логика ролей

## Узлы

| Узел | IP | Роль | Активные cron |
|------|-----------|--------|-------------------------------|
| PBX1 | 172.19.3.202 | MASTER | sync_from_202, sync_freepbx_config, pbx_failover |
| PBX2 | 172.19.3.203 | BACKUP | sync_from_202, sync_freepbx_config, pbx_failover |

**На обеих нодах cron одинаковый.** Поведение определяется наличием IP `172.19.3.202`:

- если на узле есть IP **172.19.3.202** → он автоматически **MASTER**;
- если нет → это **BACKUP**.

Такая схема защищает от «двойного мастера»: каждый скрипт в начале проверяет,
поднят ли VIP локально, и если да — синхронизация с мастера не выполняется
(мастер не тянет данные сам с себя).

## Схема

```
             VIP 172.19.3.202  (рабочий IP телефонии)
                      │
             ┌────────┴────────┐
             │   SIP-провайдеры │
             └────────┬────────┘
                      │
        ┌─────────────▼─────────────┐
        │  ACTIVE MASTER  .202       │
        └─────────────┬─────────────┘
                      │  синхронизация →
        ┌─────────────▼─────────────┐
        │  BACKUP NODE    .203       │
        └───────────────────────────┘
```

## Как определяется роль (внутри скриптов)

```bash
LOCAL_IP=$(ip -4 addr show eth0 | awk '/inet /{print $2}' | cut -d/ -f1 | head -n1)

# VIP поднят локально → это MASTER → выходим (синх не нужен)
if ip -4 addr show eth0 | grep -q " 172.19.3.202/"; then
    exit 0
fi
```

`pbx_failover.sh` отдельно считает адреса на интерфейсе:

- есть только `172.19.3.202` (и нет `.203`) → узел уже MASTER, действий нет;
- есть `172.19.3.203` → узел BACKUP, идёт проверка доступности мастера.

## Что синхронизируется

| Данные | Источник → приёмник | Скрипт | Частота |
|---|---|---|---|
| CDR / CEL (история звонков) | .202 → .203 | sync_from_202.sh | каждые 5 мин |
| Записи разговоров (monitor) | .202 → .203 | sync_from_202.sh | каждые 5 мин |
| Таблицы конфигурации FreePBX | .202 → .203 | sync_freepbx_config.sh | каждый час |
| Custom-звуки (`sounds/ru/custom`) | .202 → .203 | sync_freepbx_config.sh | каждый час |
| Всё обратно (при возврате роли) | .203 → .202 | pbx_reverse_sync.sh | вручную |

## Итоговая схема ролей

```
Нормальная работа:
    202 — MASTER
    203 — BACKUP

Failover:
    202 падает
    203 → MASTER (забирает VIP 172.19.3.202)

Восстановление 202:
    присваиваем ему IP 203 → он автоматически BACKUP

Ручной возврат:
    отключить 203
    поставить IP 202 → MASTER
    203 → BACKUP (после включения)
```
