# Аварийные сценарии и возврат ролей

## 1. Падение MASTER (.202)

Failover **автоматический** — действий не требуется. `pbx_failover.sh` на .203
через 2 неудачные проверки заберёт VIP `172.19.3.202`, восстановит шлюз, сделает
ARP refresh и пришлёт уведомление в Telegram.

Проверить: `ip addr show eth0` на .203 должен показывать `172.19.3.202/24`.

## 2. Восстановление гипервизора/ноды PBX1

После восстановления PBX1 **не должен** сразу поднимать `172.19.3.202` — иначе будет
два мастера. Верните его как BACKUP:

```
1. Выключить NIC
2. Назначить IP 172.19.3.203
3. Перезапустить сеть
4. Включить NIC
```

Нода автоматически станет BACKUP и начнёт тянуть данные с текущего мастера (.203).

## Ручной возврат ноды в BACKUP (IP 172.19.3.203)

Если нода перешла в MASTER и держит `172.19.3.202`, её можно вернуть в BACKUP:

```bash
# Очистить все IP интерфейса
ip addr flush dev eth0

# Назначить основной BACKUP-адрес
ip addr add 172.19.3.203/24 dev eth0

# Указать шлюз по умолчанию
ip route add default via 172.19.3.1 dev eth0
```

После этого:
- нода работает под IP `172.19.3.203`;
- failover-скрипт снова считает её BACKUP;
- синхронизация конфигурации с мастером (.202) включена.

## 3. Плановый возврат роли MASTER на PBX1

Порядок, чтобы не потерять данные, накопленные на .203 за время аварии:

```
1. Отключить 203 (снять VIP)
2. На 202 назначить IP 172.19.3.202
3. Включить NIC
4. На 203 выполнить /root/pbx_reverse_sync.sh   ← ДО того, как .203 станет BACKUP,
                                                   пока VIP ещё на нём
```

> ⚠️ `pbx_reverse_sync.sh` работает только когда VIP `172.19.3.202` поднят локально.
> Запускать его нужно на узле, который сейчас является действующим мастером и с которого
> возвращаются данные.

## Полезные команды

```bash
ip addr show eth0                                   # какие IP на интерфейсе
ping -c 5 172.19.3.202                              # доступность мастера
ssh -i /root/.ssh/id_rsa root@172.19.3.202 echo ok # SSH до мастера
tail -n 50 /var/log/pbx_failover.log               # лог failover
ip neigh flush all                                  # сброс ARP-кэша
cat /var/run/pbx_failover_failcount                # текущий счётчик неудач
```
